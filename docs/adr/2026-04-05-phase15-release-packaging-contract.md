# ADR: Phase 15 Release Packaging Contract

- Status: Accepted
- Date: 2026-04-05

## Context

Roadmap Phase 15 requires distribution-ready binary artifacts, install/upgrade paths, and reproducible release metadata.

The repository already had:

- cross-compile targets in `packages/opencode-core/Makefile`
- root orchestration via `Makefile`
- basic install/upgrade/uninstall scripts

What was missing for release packaging depth:

- packaged artifact directory at root release surface
- checksum manifest for verification
- machine-readable release manifest for tooling

## Decision

Adopt a packaging contract with these outputs under `dist/releases/`:

1. release binaries (`dh-darwin-arm64`, `dh-darwin-amd64`, `dh-linux-amd64`, `dh-linux-arm64`)
2. `SHA256SUMS` generated by `shasum -a 256`
3. `manifest.json` containing:
   - `generatedAt`
   - per-file `name`
   - per-file `sha256`
   - per-file `sizeBytes`

Implementation:

- add `scripts/package-release.sh` to collect binaries and emit checksums + manifest
- wire root `Makefile` target `package-release`
- make root `release-all` call `package-release` after cross-compile steps
- extend `scripts/install.sh` with optional checksum verification argument

## Consequences

Positive:

- release outputs are verifiable and tool-friendly
- install path can enforce checksum when expected hash is provided
- packaging can run from one command (`make release-all`)

Trade-offs:

- checksum/manifest generation currently uses shell tooling and assumes `shasum` availability
- signing/notarization is still out of scope and remains follow-up work

## Verification

Validated on 2026-04-05:

- `make release-all` passes
- `dist/releases/SHA256SUMS` exists
- `dist/releases/manifest.json` exists and includes all 4 target binaries
- packaged binary runs (`dist/releases/dh-darwin-arm64 --help`)
