# TUI MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a dependency-free terminal UI MVP that attaches to the DH server/SDK contract.

**Architecture:** Keep the first TUI as testable TypeScript modules: a reducer owns state, a renderer owns terminal text, a controller owns SDK/server interaction, and the CLI command wires attach/start modes. Use Node `readline/promises` only at the entry point.

**Tech Stack:** TypeScript ESM, Vitest, Node `readline/promises`, existing `packages/server` and `packages/sdk`.
